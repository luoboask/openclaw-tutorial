# OpenClaw Memory 功能深度解析

> 让 AI 拥有长期记忆，打造真正个性化的助手体验

---

## 为什么需要 Memory？

传统 AI 对话是**无状态**的——每次对话都是全新的开始。Memory 让 AI 能够：

- 🧠 **记住你的偏好** - 喜欢的回复风格、常用工具
- 📅 **追踪项目进度** - 长期任务的上下文
- 👤 **了解你的习惯** - 工作模式、沟通方式
- 📝 **积累知识** - 从你的对话中学习

---

## 一、Memory 系统架构

### 三层记忆模型

```
┌─────────────────────────────────────┐
│  长期记忆 (Long-term Memory)         │
│  - 重要事实、偏好设置                 │
│  - 持久存储，跨会话保留               │
├─────────────────────────────────────┤
│  中期记忆 (Working Memory)           │
│  - 当前项目上下文                     │
│  - 几小时到几天的活动                │
├─────────────────────────────────────┤
│  短期记忆 (Short-term Memory)        │
│  - 当前对话上下文                     │
│  - 单次会话内有效                     │
└─────────────────────────────────────┘
```

---

## 二、配置 Memory 系统

### 基础配置

```json
// ~/.openclaw/openclaw.json
{
  "memory": {
    "enabled": true,
    "provider": "local",
    "local": {
      "path": "~/.openclaw/memory"
    }
  }
}
```

### 使用 Ollama 本地嵌入

```json
{
  "memory": {
    "enabled": true,
    "provider": "ollama",
    "ollama": {
      "baseUrl": "http://localhost:11434",
      "model": "nomic-embed-text"
    }
  }
}
```

## 三、Memory 使用方法

### 自动记忆

OpenClaw 会自动提取对话中的重要信息：

```
用户: 我明天下午3点有个会议
AI: [自动记录到中期记忆]

用户: 我偏好简洁的回复
AI: [自动记录到长期记忆]
```

### 手动管理

```bash
# 查看记忆
openclaw memory search "项目进度"

# 添加记忆
openclaw memory add "用户的邮箱是 user@example.com"

# 删除记忆
openclaw memory delete <memory-id>

# 清空记忆
openclaw memory clear
```

---

## 四、记忆检索机制

### 语义搜索

Memory 使用**向量嵌入**实现语义搜索：

```
用户问: "我昨天说的那个项目怎么样了？"

Memory 系统:
1. 将问题转换为向量
2. 在记忆库中搜索相似向量
3. 返回相关记忆:
   - "用户正在开发 RedClaw 项目"
   - "昨天完成了文章发布功能"
```

## 五、高级用法

### 记忆分类

```json
{
  "memory": {
    "categories": {
      "personal": {
        "description": "个人信息和偏好",
        "priority": "high"
      },
      "projects": {
        "description": "项目相关信息",
        "priority": "medium"
      }
    }
  }
}
```

### 记忆导入导出

```bash
# 导出记忆
openclaw memory export > memories.json

# 导入记忆
openclaw memory import < memories.json
```

## 六、隐私与安全

### 数据控制

- **本地存储**: 所有记忆默认保存在本地
- **加密选项**: 支持 AES-256 加密
- **访问控制**: 可设置记忆访问权限

---

## 七、使用建议

### ✅ 推荐做法

1. **定期整理** - 每月清理不重要的记忆
2. **分类管理** - 使用类别组织记忆
3. **敏感信息** - 谨慎存储密码等敏感数据
4. **备份重要记忆** - 定期导出备份

### ❌ 避免做法

1. **存储敏感密码** - 使用专门的密码管理器
2. **无限增长** - 设置记忆上限
3. **依赖短期记忆** - 重要信息显式记录
